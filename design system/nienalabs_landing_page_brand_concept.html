Concept Overview
The Story We're Telling
This landing page is not a brochure. It is a manifesto that unfolds as the visitor scrolls — a journey from the darkness of unbuilt potential, through the act of creation, to the illuminated proof of what software can make possible. Every section has a purpose in the narrative arc.
Tone
Gravitas. Controlled ambition. The confidence of an institution that knows it builds for centuries, not quarters.
Emotion
Awe at the scale of what software can achieve. Urgency for founders with an idea. Trust for enterprises choosing a partner.
Narrative
Darkness → Spark → Creation → Proof → Invitation. The page is a story. Scroll is the chapter turn.

Section 03
The Philosophy — A Scroll-Driven Reveal
This section exists to earn trust before asking for anything. It tells the reader who Nienalabs is at its core — not a feature list, not a service menu, but a philosophical stance. Rendered as large editorial text that assembles word by word as the user scrolls.
Philosophy Section
Our Philosophy
We believe the world can be changed the way we found it.
Every product we build has one purpose:
to push humanity forward.
Layout
Full-width section, --color-void background. Centered content column, max-width 760px. Tall vertical padding: --space-11 (128px) top and bottom. This section demands space.
Typography
The philosophy text renders in three parts: (1) "We believe the world can be changed the way we found it." — Cormorant Garamond Light, 48px, Parchment. (2) "Every product we build has one purpose:" — same font, 36px, Vellum (secondary text). (3) "to push humanity forward." — same font, italic, 48px, Gold (#c9a84c). The gold italic line is the emotional peak of the entire page.
Image Placement
No photographic images in this section. The only visual is the typography itself and a single horizontal gold rule (1px, 80px wide) between the overline and the text block. Negative space is the luxury signal here.
GSAP ScrollTrigger
Each word of the philosophy text is wrapped in a span. On ScrollTrigger, words transition from color: rgba(232,223,200,0.12) (barely visible, like ghost text carved in stone) to their final color, staggered 0.03s per word. The effect: the philosophy text "illuminates" as the user reads it, word by word, as if a light is traveling across a wall inscription.
Vertical Separator
At the bottom of this section, a 1px gold vertical line, 64px tall, fades in and "grows" downward (scaleY: 0 → 1, transform-origin top) as the section exits. It connects visually to the next section, implying continuation.

Section 05
The Process — How We Build
A horizontal scroll-driven timeline (on desktop) that shows the Nienalabs approach to building software. Not a generic "Discover → Design → Develop → Deploy" diagram. A narrative of how great software comes to life. On mobile, this becomes a vertical stacked sequence.
Layout
Full-width horizontal panel, viewport height ~80vh. Left side (40%): sticky text block with overline, section title, and a paragraph explaining the process philosophy. Right side (60%): horizontal scroll container with 5 steps, each ~320px wide, 24px gutter. As the user scrolls vertically, the right side scrolls horizontally — achieved via GSAP ScrollTrigger horizontal: true pinning the section.
Each Step Card
320×440px card. --color-slate-deep background. Gold top-border on the "active" (currently centered) step. Cinzel overline with a step number (01, 02, 03, 04, 05) in Cormorant Garamond 64px Light, gold, 15% opacity — this giant number floats behind the title. Then: step name in Cinzel 11px, then Cormorant 26px title, then EB Garamond 15px body copy of 3–4 sentences. A horizontal gold line (40px) between the number zone and the text.
5 Steps — Copy
01 — Understand: "We begin with the problem, not the stack." 02 — Architect: "Every line of code is preceded by a decision." 03 — Build: "Precision. Iteration. No shortcuts." 04 — Launch: "Deployment is not the finish line." 05 — Evolve: "The best software is always becoming."
GSAP Horizontal Scroll
Section is pinned while scroll progress drives x translation of the card container. Cards at x: 0 on entry, travel to x: -(totalWidth - viewportWidth) by section exit. Each card "brightens" (border-top gold fades in, body text opacity increases from 0.4 → 1) when it reaches the center viewport position. Progress indicator: a thin gold horizontal line at the bottom, scaleX: 0 → 1 as scroll advances.
Image — Step Cards
No photographs in these cards. The large faded step number is the only "visual." This restraint says: our process is intellectual, not performative. The typography IS the illustration.

Section 07
Selected Work — The Portfolio
A curated selection of 2–3 projects. Not a gallery. A curated exhibition — each project given breathing room, positioned asymmetrically, with one large image and deliberate editorial copy. This is Nienalabs showing, not telling.
Layout — Each Project
Alternating left-right layout. Project 1: image takes 58% of width on the left (full bleed image, 480px tall, object-fit:cover, 4px border-radius, scrim applied), copy takes 38% on right (vertically centered). Project 2: copy on left, image on right. 64px vertical gap between projects. No cards — the image IS the card.
Image Requirements
Fine art photography style per brand imagery rules: warm tones, muted palette, architectural composition, soft grain, dramatic raking light. Subject matter: real or analogous representations of the product in use — abstracted, not literal screenshots. Each image: golden hour warm toning, sepia wash overlay at 15% opacity applied via CSS mix-blend-mode: multiply on a rgba(201,168,76,0.12) overlay div.
Project Copy Structure
Cinzel overline (9px, gold): project category (e.g., "Enterprise Web Application"). Cormorant Garamond 36px title: the project name or tagline. EB Garamond 16px body: 3 sentences about the challenge and what was built. A Cinzel 9px tag row for tech stack (e.g., "React · Node.js · AWS · PostgreSQL"). Finally: a ghost button "Read Case Study" (transparent, subtle border, gold text hover).
GSAP ScrollTrigger
Each project pair enters with a parallax: the image scrolls at 0.85x speed (GSAP scrub: true, y: -40 → 40 as the section scrolls through), while the text scrolls at normal speed. This creates depth — the image feels cinematic and dimensional. Text block: x: 30, opacity: 0 → x: 0, opacity: 1 on entry, 0.8s ease. Image: scale: 1.05 → 1 on entry (Ken Burns-style settle), 1.2s.
Section Header
Before the projects: Cinzel overline "Selected Work", Cormorant 48px title "Built with purpose.", 40px gold horizontal line, then EB Garamond body "A selection of products we're proud to have built." — 2 sentences max. 96px below the section header, the first project begins.

Animation Reference
GSAP Master Scroll Map
All animations use GSAP with ScrollTrigger. No CSS-only animations for scroll-driven effects. The following eases and principles govern all motion.
Primary Ease
power2.out for entrances. power1.inOut for scrub/parallax. Never bounce, elastic, or spring on structural elements.
Text Reveals
Word-by-word: GSAP stagger on span wraps. opacity: 0 → 1, stagger 0.025s per word. For hero headline: SplitText by chars for more drama.
Parallax Depth
Images move at 0.85x scroll speed. Background geometry at 0.92x. Text at 1x. This creates 3 depth planes.
Pinned Sections
Section 05 (Process): pinned with horizontal scroll. scrub: 1.5 for smooth, lag-behind feel. Duration: scrollTrigger: { scrub: true, end: "+=2000" }.
Line Draws
Gold decorative lines use scaleX: 0 → 1 or scaleY: 0 → 1, transform-origin start. Duration 0.5–0.8s. These punctuate section entries.
Counter Animation
Section 06 stats: gsap.to(obj, { val: targetNumber, duration: 1.5, ease: 'power2.out', onUpdate: () => el.textContent = Math.round(obj.val) }).
Reduced Motion
All animations must check window.matchMedia('(prefers-reduced-motion: reduce)'). If true: skip all scroll-triggered animations. Content is visible at full opacity by default. Transitions max 200ms.
Performance Rule
All animated properties must be transform or opacity only. Never animate width, height, top, left, margin, padding, or background-color — these trigger layout/paint.
